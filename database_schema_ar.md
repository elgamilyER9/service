# توثيق قاعدة البيانات لمشروع الخدمات (Service Project)

هذا الملف يحتوي على شرح مفصل للجداول، الحقول، والعلاقات في قاعدة البيانات باللغة العربية.

---

## 1. جدول المستخدمين (`users`)
يخزن معلومات جميع مستخدمي النظام (عملاء، فنيين، مدراء).

| الحقل | الوصف |
| :--- | :--- |
| `id` | المعرف الفريد للمستخدم |
| `name` | اسم المستخدم |
| `email` | البريد الإلكتروني (فريد) |
| `password` | كلمة المرور (مشفرة) |
| `role` | دور المستخدم (مثلاً: عميل، فني، مدير) |
| `phone` | رقم الهاتف |
| `address` | العنوان |

**العلاقات:**
- **لديه العديد من طلبات الخدمة** (`hasMany` ServiceRequests).
- **لديه ملف فني واحد** (`hasOne` Technician) - في حال كان دوره "فني".
- **لديه العديد من التقييمات** (`hasMany` Reviews).

---

## 2. جدول الفنيين (`technicians`)
يحتوي على البيانات الإضافية الخاصة بالمقدمين للخدمة (الفنيين).

| الحقل | الوصف |
| :--- | :--- |
| `id` | المعرف الفريد للفني |
| `user_id` | معرف المستخدم المرتبط (علاقة مع جدول users) |
| `category_id` | معرف القسم التابع له (علاقة مع جدول categories) |
| `experience_years` | عدد سنوات الخبرة |
| `hourly_rate` | سعر الساعة |
| `availability` | المواعيد المتاحة |
| `city` | المدينة |
| `bio` | نبذة عن الفني |
| `rating` | التقييم العام |
| `is_available` | حالة التوفر الحالية (متاح/غير متاح) |

**العلاقات:**
- **ينتمي إلى مستخدم** (`belongsTo` User).
- **ينتمي إلى قسم** (`belongsTo` Category).
- **لديه العديد من طلبات الخدمة** (`hasMany` ServiceRequests).
- **لديه العديد من التقييمات** (`hasMany` Reviews).

---

## 3. جدول الأقسام (`categories`)
يصنف الخدمات والفنيين (مثل: سباكة، كهرباء، نجارة).

| الحقل | الوصف |
| :--- | :--- |
| `id` | المعرف الفريد للقسم |
| `name` | اسم القسم (مثلاً: كهرباء) |
| `description` | وصف القسم |
| `image` | صورة القسم |

**العلاقات:**
- **لديه العديد من الخدمات** (`hasMany` Services).
- **لديه العديد من الفنيين** (`hasMany` Technicians).

---

## 4. جدول الخدمات (`services`)
يحتوي على تفاصيل الخدمات المحددة المتاحة للطلب.

| الحقل | الوصف |
| :--- | :--- |
| `id` | المعرف الفريد للخدمة |
| `name` | اسم الخدمة |
| `description` | وصف الخدمة |
| `price` | السعر التقديري للخدمة |
| `estimated_time` | الوقت التقديري لإنجاز الخدمة |
| `category_id` | القسم التابع له (علاقة مع جدول categories) |
| `image` | صورة توضيحية للخدمة |

**العلاقات:**
- **ينتمي إلى قسم** (`belongsTo` Category).
- **لها العديد من طلبات الخدمة** (`hasMany` ServiceRequests).

---

## 5. جدول طلبات الخدمة (`service_requests`)
يسجل جميع الطلبات التي يقوم بها العملاء.

| الحقل | الوصف |
| :--- | :--- |
| `id` | المعرف الفريد للطلب |
| `user_id` | العميل الذي طلب الخدمة (علاقة مع users) |
| `technician_id` | الفني المكلف بالمهمة (علاقة مع technicians) |
| `service_id` | الخدمة المطلوبة (علاقة مع services) |
| `status` | حالة الطلب (قيد الانتظار، مقبول، مكتمل، ملغي) |
| `requested_date` | التاريخ المطلوب |
| `requested_time` | الوقت المطلوب |
| `address` | عنوان تنفيذ الخدمة |
| `phone` | رقم هاتف للتواصل |
| `notes` | ملاحظات العميل |

**العلاقات:**
- **ينتمي إلى مستخدم** (`belongsTo` User).
- **ينتمي إلى فني** (`belongsTo` Technician).
- **ينتمي إلى خدمة** (`belongsTo` Service).

---

## 6. جدول التقييمات (`reviews`)
يخزن تقييمات العملاء للفنيين بعد إتمام الخدمة.

| الحقل | الوصف |
| :--- | :--- |
| `id` | المعرف الفريد للتقييم |
| `service_request_id` | معرف الطلب المرتبط (علاقة مع service_requests) |
| `user_id` | معرف المستخدم الذي قام بالتقييم |
| `technician_id` | معرف الفني الذي تم تقييمه |
| `rating` | التقييم الرقمي (مثلاً من 1 إلى 5) |
| `comment` | التعليق النصي |

**العلاقات:**
- **ينتمي إلى مستخدم** (`belongsTo` User).
- **ينتمي إلى فني** (`belongsTo` Technician).
- **ينتمي إلى طلب خدمة** (`belongsTo` ServiceRequest).
